# Analysis of Algorithms

Analysis of Algorithms is a course (or subject area) focused on how efficient algorithms are — not just whether a program works, but how well it works in terms of:

*   speed (time complexity)
*   memory usage (space complexity)
*   scalability as inputs grow

It sits at the intersection of:
*   computer science
*   discrete mathematics
*   probability
*   logic
*   mathematical proof techniques

The topics listed below are the mathematical foundations that support algorithm analysis.

---

## Core Goal of the Course

The course teaches you to answer questions like:
*   Which algorithm is faster?
*   How does performance change with millions of inputs?
*   Can this problem be solved efficiently?
*   What is the theoretical limit of computation?
*   How can randomness improve algorithms?

---

## What the Listed Subjects Mean

### Data Structures
How data is organized efficiently.
*   **Examples:** arrays, linked lists, stacks, queues, trees, hash tables, graphs
*   **Why it matters:** The choice of data structure dramatically affects algorithm speed.
*   **Example (Searching):**
    *   unsorted list → slow
    *   hash table → fast

### Probability
Used for:
*   randomized algorithms
*   average-case analysis
*   machine learning foundations
*   hashing
*   performance estimation
*   **Example:** A randomized quicksort algorithm often performs better in practice.

### Combinatorics
Study of counting and arrangements.
Used for:
*   graph algorithms
*   complexity analysis
*   optimization problems
*   recursion analysis
*   **Example:** “How many possible paths exist?” or “How many combinations must an algorithm check?”

### Theoretical Computer Science
The mathematical theory behind computation.
Topics may include:
*   automata
*   computability
*   complexity theory
*   NP-complete problems
*   **Key Learning:** What computers can solve and what they cannot solve efficiently.

### Calculus
Used less heavily than discrete math, but still important for:
*   continuous optimization
*   machine learning
*   numerical analysis
*   growth-rate approximations
*   **Example:** Analyzing functions and asymptotic growth.

### Mathematical Modeling
Turning real-world problems into mathematical systems.
*   **Examples:** traffic routing, network optimization, scheduling, prediction systems
*   **Application:** Algorithms solve these models efficiently.

### Applied Mathematics
Practical use of mathematical methods in computing and engineering.
This connects theory to:
*   simulations
*   optimization
*   AI
*   cryptography
*   operations research

### Algorithms
Step-by-step procedures for solving problems.
*   **Examples:** sorting, searching, shortest path, dynamic programming, divide and conquer
*   **Focus:** This is the heart of the course.

---

## Main Skills You Learn

### Analyze efficiency
Using:
*   Big O notation
*   time complexity
*   space complexity
*   **Example:** O(nlogn)

### Design algorithms
Methods like:
*   recursion
*   greedy algorithms
*   dynamic programming
*   graph traversal
*   backtracking

### Prove correctness
You mathematically prove:
*   the algorithm works
*   it always terminates
*   it gives the optimal answer

### Compare computational complexity
You classify problems into categories like:
*   polynomial time
*   exponential time
*   NP-hard
